imblend

融合 2 幅图像

简介

I = imblend(fg, bg)
I = imblend(fg, bg, mask)
I = imblend(___, Name, Value)

用法

I = imblend(fg, bg) 使用 Alpha 混合将前景图像 fg 与背景图像 bg 进行融合。
I = imblend(fg, bg, mask) 仅将前景图像中由 mask 指定的区域与背景图像进行融合。
I = imblend(___, Name, Value) 支持任何上述语法，通过一个或多个可选的名称-值参量，精细化调整图像融合效果。

参数说明

输入参数

fg — 前景图像
二维灰度图像 | 二维 RGB 图像

前景图像，指定为二维灰度图像或二维 RGB 图像。
前景图像允许与背景图像尺寸不一致。融合时，函数仅取两幅图像重叠的前导区域：即取前 min(height_fg, height_bg) 行与前 min(width_fg, width_bg) 列进行融合。

数据类型：single | double | int16 | uint8 | uint16

bg — 背景图像
二维灰度图像 | 二维 RGB 图像

背景图像，指定为二维灰度图像或二维 RGB 图像。

数据类型：single | double | int16 | uint8 | uint16

mask — 用于融合的前景像素掩码
true 或 1（默认）| false 或 0 | 逻辑矩阵

用于指定前景图像中需要融合的像素区域，指定为 true 或 1（默认）、false 或 0，与前景图像尺寸相同的逻辑矩阵。

数据类型：logical

名称-值参数

将可选的参量对组指定为Name1, Value1, ... , NameN, ValueN，其中 Name 是参量名称，Value 是对应的值。名称-值参量必须出现在其他参量后，但对各个参量对组的顺序没有要求。
示例：imblend(F, B, "Location", [10 12])

Mode — 混合模式
"Alpha"（默认）| "Min" | "Max" | "Average" | "Overlay"

混合模式，指定为"Alpha"，"Min"，"Max"，"Average" 或 "Overlay"。

混合模式 算法 应用场景
"Alpha" - 线性结合前景与背景图像。
- 使用前景不透明度确定线性组合的权重。
- 创建部分透明效果。		 增强现实、计算机生成图形、多模态医学影像可视化。
"Min" - 使用最小强度投影。
- 混合图像保留前景与背景图像中强度最小的像素。
- 保留前景和背景图像中最暗的区域。		 天体摄影（降噪）、安防应用（增强暗区细节）。
"Max" - 使用最大强度投影。
- 混合图像保留前景与背景图像中强度最大的像素。
- 保留前景和背景图像中最亮的区域。		 医学影像（突出特征）、监控（检测图像序列中的运动或变化）。
"Average" - 混合图像保留前景与背景图像强度的平均值。 摄影降噪（通过对同一场景多张图像取平均减少随机噪声）。
"Overlay" - 根据像素亮度选择乘法或滤色混合方式。
- 保留背景图像的高光和阴影，同时融合前景图像的纹理。		 数字艺术、照片编辑（添加纹理、调整光照、通过混合纹理/图案与原图创建特效）。

数据类型：char | string

Location — 前景图像左上角在背景图像中的混合起始坐标
[1 1]（默认）| 1×2 数值向量

前景图像左上角在背景图像中的混合起始坐标，指定为 1×2 数值向量。
向量第一个元素表示背景图像中的 x 坐标（列索引），第二个元素表示 y 坐标（行索引）。

数据类型：single | double

FilterSize — 导向混合中的滤波器大小
[3 3]（默认）| 二元正整数向量

导向混合中滤波器的大小，指定为二元正整数向量。增大滤波器尺寸可增强掩膜边缘的平滑效果。

数据类型：single | double | int8 | int16 | int32 | int64 | uint8 | uint16 | uint32 | uint64

输出参数

I — 融合后的图像
二维灰度图像 | 二维 RGB 图像

融合后的图像，以二维灰度图像或 RGB 图像形式返回，其尺寸和数据类型与背景图像相同。
