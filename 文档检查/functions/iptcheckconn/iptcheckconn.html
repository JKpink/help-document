iptcheckconn

验证连通性参数的合法性

简介

iptcheckconn(conn, func_name, var_name, arg_pos)

用法

iptcheckconn(conn, func_name, var_name, arg_pos) 检查 conn 是否为有效的像素连接，并在连接无效时发出格式化的错误消息。

  • 如果连接有效，则 iptcheckconn 不返回。有效的连接是这些标量值之一：1，4，6，8，18或26。连接也可以是一个由0 和 1 的 3x3x ... x 3 数组构成。连接数组的中心元素必须非零，并且数组必须关于其中心对称。
  • 如果连接无效，则 iptcheckconn 将发出格式化的错误消息，其中包括有关函数名称、变量名称和参数位置的信息。这些值仅用于创建错误消息，而不用于检查像素连接是否有效。

参数说明

输入参数

conn — 像素连接
数字标量 |数值数组

要检查的像素连接，指定为数值标量或数组。

数据类型： single | double

func_name — 函数名称
字符向量 |字符串标量

当 conn 为无效像素连通时，包含在错误消息中的函数名称，指定为字符向量或字符串标量。

数据类型： char | string

var_name — 变量名称
字符向量 |字符串标量

当 conn 为无效像素连通时，包含在错误消息中的变量名称，指定为字符向量或字符串标量。

数据类型： char | string

arg_pos — 参数位置
正整数

当 conn 为无效像素连通时，包含在错误消息中的参数位置，指定为数值标量。
